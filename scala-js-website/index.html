Correctness

Strong typing guarantees your code is free of silly mistakes; no more mixing up strings or numbers, forgetting what keys an object has, or worrying about typos in your method names. Scala.js takes care of all this tedious book-keeping for you, letting you focus on the actual, more interesting problem your website is trying to solve.

Performance

Scala.js compiles Scala code into highly efficient JavaScript utilizing an internal optimizer. Incremental compilation guarantees speedy (1-2s) turn-around times when your code changes. Generated JavaScript is both fast and small, starting from 45kB gzipped for a full application.

Interoperability

Use jQuery or any HTML5 feature directly from your Scala.js code, either in a typed or untyped way. All JavaScript libraries work with Scala.js code. If you like React.js, or Angular.js, that's great! You can keep using them while getting the benefits of Scala.js

Build amazing applications fast!

Your Scala.js application can utilize the full extent of HTML5, CSS3 and all existing JavaScript libraries, including Angular and React. Focus on writing clean and error-free code with the help of Scala's strong type system and you will be delivering in no time! Less hassle, more productivity with Scala's excellent libraries, fully at your disposal even on the browser.

First class IDE support

Imagine a world where your IDE can deliver 100% accurate code completion 100% of the time for 100% of the libraries. All typos and type errors are automatically caught as you write your code. Refactor any field or method name with ease and confidence. All this and more is available to you in the Scala.js development environment.

Go beyond Javascript ES6, today

Scala language already has all the new features coming in ES6 and much more.

Classes
JavaScript
var Person = function(firstName) {
    this.firstName = firstName;
};

Person.prototype.walk = function() {
    console.log("I am walking!");
};

Person.prototype.sayHello = function(target) {
    console.log("Hello " + target +
                ", I'm " + this.firstName);
};
Scala.js
class Person(firstName: String) {
  def walk = println("I am walking!")

  def sayHello(target: String) =
    println(s"Hello $target, I'm $firstName")
}
Fat arrow functions
JavaScript
var names = persons.map(function(p) {
    return p.firstName
});
Scala.js
val names = persons.map(p => p.firstName)
// or an even shorter version
val names = persons.map(_.firstName)
Collections
JavaScript
var personMap = {
    "10": new Person("Roger Moore"),
    "20": new Person("James Bond")
};
var names = [];
for(var key in personMap) {
    if(Object.prototype.hasOwnProperty.call(names, key)) {
        if(parseInt(key) > 15) {
            names.push(
                key + " = " + personMap[key].firstName
            );
        }
    }
}
Scala.js
val personMap = Map(
  10 -> new Person("Roger Moore"),
  20 -> new Person("James Bond")
)
val names = for {
  (key, person) <- personMap
  if key > 15
} yield s"$key = ${person.firstName}"

Safer, Better JavaScript

Several variants of plain JavaScript (ES5) exist today, most notably ES6 and TypeScript. If you're already going to compile something to Javascript, why not compile something nicer? 

js> ["10", "10", "10", "10"].map(parseInt)
[10, NaN, 2, 3] // WTF
scala> List("10", "10", "10", "10").map(parseInt)
List(10, 10, 10, 10) // Yay!

While Typescript and ES6 make your code safer and more modern, they still leave many of Javascript's [Gotcha](https://en.wikipedia.org/wiki/Gotcha_(programming))s wide-open for you to trip on. Scala.js provides all the same awesome things that Typescript or ES6/ES7 provide, but better: more modern than ES6, more safe than Typescript, on top of fixing many of the silly issues that makes programming in Javascript frustrating.
